Simple Widget Tester

This page demonstrates how the widget can be embedded on a customer's website using server-side configuration.

Note: The widget configuration is stored on the server and loaded dynamically when the widget initializes.

Production Widget

This is the production widget with ID: pjxZqkQ9fAZaqIoOOJJx

<script src="https://us-central1-adealo-ce238.cloudfunctions.net/widget?widgetId=pjxZqkQ9fAZaqIoOOJJx"></script>
<chat-widget></chat-widget>

Local Development Widget

This is the local development widget with ID: pjxZqkQ9fAZaqIoOOJJx

<script src="http://localhost:5001/adealo-ce238/us-central1/widget?widgetId=pjxZqkQ9fAZaqIoOOJJx"></script>
<chat-widget></chat-widget>

To test with local development server, uncomment the script tag in the HTML.

Important: Make sure you have created a widget configuration in Firestore with the widget ID before testing.

How It Works

When the page loads, the widget script:

  1. Extracts the widgetId from the script URL
  2. Fetches the widget configuration from the server using the widgetId
  3. Caches the configuration in localStorage for future use
  4. Renders the widget with the configuration

Benefits of Server-Side Configuration

  • Security: The configuration is stored on your server, not in the customer's HTML.
  • Scalability: You can update the configuration for all customers without requiring them to change their embed code.
  • Privacy: Sensitive data (e.g., customerId) is not exposed in the HTML.
  • Dynamic Updates: You can update the configuration in real-time (e.g., change colors, enable/disable features).
